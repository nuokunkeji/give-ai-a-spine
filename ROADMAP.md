# Roadmap

The target is 1,000 GitHub stars in 60–90 days. This is a growth objective, not a promise or a substitute for user value.

## Week 1 — v0.1.0

- Publish one-sentence positioning, paired raw evaluations, demo GIF, and three strong examples.
- Verify clean-session installation in Codex and Claude Code.
- Target: 100 stars in seven days.

## Week 2 — v0.1.1

- Fix real installation problems.
- Add community-submitted sycophancy and overcorrection cases.
- Do not add features merely to create release activity.

## Week 3 — v0.2.0

- Improve explicit audit mode.
- Add adapters only where real agent behavior differs.
- Re-run the public paired evaluation before release.

## Week 4 — community challenge

- Invite before/after submissions under “Give your AI a spine.”
- Submit relevant awesome-list pull requests after maintainer approval.
- Publish only to communities the repository owner explicitly authorizes.

## Reality check

After at least five substantive community launches, if the repository has fewer than 50 stars on day seven, revisit positioning, demo clarity, and the first-run result before building more features.
