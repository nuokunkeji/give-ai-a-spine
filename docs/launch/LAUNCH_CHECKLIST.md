# Launch checklist

External publication requires explicit repository-owner approval.

- [ ] Confirm `v0.1.0` release and CI are green.
- [ ] Re-run clean-session Codex and Claude Code smoke tests.
- [ ] Verify README GIF and all raw-eval links from an incognito browser.
- [ ] Capture the release date, baseline stars, and traffic source.
- [ ] Publish to one community at a time; answer substantive questions before moving on.
- [ ] Complete at least five meaningful launches before judging the seven-day threshold.
- [ ] At day seven, record stars, clones, install issues, eval-case contributions, and conversion by source.
- [ ] If stars are below 50, rewrite positioning and the demo before adding features.
