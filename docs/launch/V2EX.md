# V2EX 首发草稿

## 标题

[开源] Spine：给 Codex / Claude Code / Cursor 一套“别迎合我”的共同修正协议

## 正文

做这个项目的原因很简单：很多 AI 在真正重要的地方，太容易顺着提问者的前提继续说下去。表达越自信，它越容易把未经验证的判断当事实。

Spine 是一个跨 Codex、Claude Code、Cursor、Gemini CLI 和 OpenCode 的 `reality-check` 技能。遇到重要决策时，它会澄清真正目标，区分事实/推断/价值/象征表达，同时检查用户的自我合理化与 AI 自己的迎合和假装确定，找到承重假设与反证，最后落到一个可观察的验证动作。

我也明确限制了“纠正过头”：不能为了显得聪明而唱反调，不能擅自心理诊断，不能嘲讽象征或玄学表达，简单问题必须直接回答。

仓库公开了 24 个测试场景、12 组 Codex 原始对照回答、评分说明和 Schema。发布门槛是重要场景平均提升至少 25 个百分点，同时简单回答不超过基线篇幅的 1.2 倍。

项目没有 MCP、账号、云服务、遥测和 npm 发布依赖。安装器遇到同名技能默认停止，`--force` 也会先备份。

希望大家提供两类反例：AI 仍然过度迎合的场景，以及 Spine 纠正过头的场景。

GitHub：https://github.com/nuokunkeji/give-ai-a-spine
